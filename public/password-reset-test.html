  • Secure token generation (1-hour expiry)

  • Professional email templates

🔧 Email Providers Supported

  • Resend (Recommended)
  • SendGrid (Enterprise)
  • SMTP (Any provider)
  • Console (Development)

Test Password Reset Flow

Step 1: Request Password Reset

Use the "Forgot Password" button on the auth page to request a reset.

POST /api/auth/forgot-password
{ "email": "user@example.com" }

Step 2: Check Email/Console

In development, the reset link appears in console. In production, it's sent via email.

http://localhost:3000/reset-password?token=...

Step 3: Reset Password

Click the link to access the password reset page with beautiful UI.

POST /api/auth/reset-password
{ "token": "...", "password": "newpassword123" }

Step 4: Success & Redirect

Success page shown, then redirect to sign in with new password.

🔧 Production Email Setup

Add one of these to your .env file:

Resend (Recommended):

RESEND_API_KEY=re_your_api_key

SendGrid:

SENDGRID_API_KEY=SG.your_api_key

SMTP (Gmail, Outlook, etc.):

SMTP_HOST=smtp.gmail.com
SMTP_USER=your@gmail.com
SMTP_PASS=your_app_password

Optional customization:

FROM_EMAIL=noreply@yourdomain.com
FROM_NAME=Your Company
BASE_URL=https://yourdomain.com
